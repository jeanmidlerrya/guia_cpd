Reiniciar os servidores

<Ordem>

  1. SRVBCKfff (Servidor Netbackup - Robot)
  2. DBfff (Servidor DB)
  3. SRVSAVEfff (Serrvidor Save)
  4. SRVAPPfff (Servidor Save Web)
  5. SRVTPLfff (Servidor Tplinux)
  6. SRVUCAfff (Servidor Rub)
  7. SRVRODCfff (Servidor A.D)

Servidor Netbackup

Para reiniciar o servodo Netbackup, abra o Putty servidor SRVBCKfff comando: sudo shutdown -r now
Aguarde uns 5 minutos e tente conectar no serviddor via putty.

Servidor DB

Para reiniciar o servidor DB abra o putty no servidor DB na tela SVDOU27 desabilita o programa FLEXJAVA, depois sair do putty abrir o navegador acessar o site srvvcbo, dentro do site selecione o servidor DBfff, botão direito do mouse em cima, Power, opção reset, para acompanhar a reinicialização, clique em launch web console.
Após reiniciar o servidor abra putty no DB execute o comando: sudo mount -a para garantir que o mapeamento está ativo, e execute o comando: sudo service dbora start.
Depois abra o putty servidor DB comando: defrun exe svdou23, colocar o progama FLEXJAVA de novo no ar.

Servidor SRVSAVE

Ainda no navegador srvvcbo Selecione o servidor srvsavefff, botão direito do mouse em cima, Power, opção reset, para acompanhar a reinicialização, clique em Launch web console. Aguarde mostrar tela de login. Abra o putty servidor srvsavefff execute o comando: sudo service slapd status, para garantir se o programa está ativo, se não está ativo execute o comnado: sudo service slapd start .

Servidor Save Web (APP)

Abra o putty servidor SRVAPPfff digite o comando: sudo service tomcat stop
Para verificar se o serviço está fora dá comando: ps -ef | grep dfrun, tem constar apenas 1 processo. Depois no navegador srvvcbo selecione o servidor (SAVEWEB) srvappfff, botão direito do mouse em cima, Power, opção reset, para acompanhar a reinicialização, clique em Launch web console. Aguarde mostrar tela de login. Depos disso abra o putty servidor srvappfff comando: sudo service tomcat status, para verificar, se esteve inactiva: sudo serivoce tomcat start para ativa o serviço.

Servidor Tplinux

Importante: Não reinicie o servidor Tplinux nem execute o (SOD) entre as 02:00hrs e 03:30hrs ou entre 04:00hrs e 05:30hrs.
Para reiniciar abra o putty servidor Tplinux no menu , selecione a opção: REINICIAR SERVIDOR TPLINUX , OPÇÃO REINICIAR SERVIDOR tecle enter! No navegador srvvcbo selecione o servidor (TPLINUX)srvtplfff , para acompanhar a reinicialização, clique em Launch web console.Aguarde mostrar tela de login.

Servidor RUB

No navegador srvvcbo Selecione o servidor (RUB)srvucafff, botão direito do mouse em cima, Power, opção reset, para acompanhar a reinicialização, clique em Launch web console. Aguarde mostrar tela de login.

Servidor A.D

No navegador srvvcbo Selecione o servidor (A.D)SRVRODCFFF, botão direito do mouse em cima, Power, opção reset, para acompanhar a reinicialização, clique em Launch web console. Aguarde carregar a interface Windows.
